##the lobotomized owl selector (茫然的猫头鹰选择器）

另一个Lobotomy Selector

像这样的选择器呢：

the-brain:empty {
   color: green;
}

什么选择器才是大脑？是我们通过MAD科学如FRANKENSTEIN’S MONSTER?又或是创建一个自定义元素或是其他的

<template>
  The brain!
</template>

var tmpl = document.querySelector('template');

var brainProto = Object.create(HTMLElement.prototype);

brainProto.createdCallback = function() {
  var root = this.createShadowRoot();
  root.appendChild(document.importNode(tmpl.content, true));
};

var brain = document.registerElement('the-brain', {
  prototype: brainProto
});

（也许该用最近出的customElements.define()？我也不知道）

如果我们按上面的这样做就能匹配最初的选择器

但是选择器表明，如果我们(的HTML)像下面这样就不能匹配：

<the-brain>
  Fallback content inside
</the-brain>

甚至这样也不匹配（译者注：标签中间有空格）

<the-brain> </the-brain>

但是我们可以用the-brain:blank{}，但:blank还不被支持。

麻木不仁的选择器

有什么比麻木不仁更恐怖？它比大部分都恐怖，这是一个至今真正奇怪的选择器，对吧？

a[href$=".zip" i] {

}

最后的i是在说明属性值”.zip”可以匹配大小写字符。

唯一正气凌然的选择器

斑马的条纹很容易实现吗？

tr:nth-child(odd) {
     background: WhiteSmoke;
}

但如果我们要消除一些行，可用JAVASCRIPT魔法给特定的行应用类名。

...
<tr><td></td></tr>
<tr class="BANISHED"><td></tr></tr>
<tr><td></td></tr>
...

.

.BANISHED {
      display: none;
}

现在我们的斑马条纹全乱了，像许多蝾螈在施法时落入大锅。（ug,对不起）

选择器级别4进行了修复：

tr:nth-child(odd of li:not(.BANISHED)) {
      background: WhiteSmoke;
}

“奇数”是仍然还会显示的行计算出来的，斑马条纹也会保留完整。

虽然还不能完全支持，那就让让我们像女妖一样尖叫到它被支持为止。

##刀穿过内脏选择器

还记得我们创建的自定义元素吗？我觉得更像是元素模板，而其中有些实际的HTML。

<template>
  <div class="braaaain">The brain!</div>
</template>

正如我们已经看到的，可以在该元素上使用CSS，它会级联进去，如你所期望的，但不能直接作用于里面的元素。

/* works fine */
the-brain {
  color: pink;
}
/* But you can't reach inside like this */
.braaaain {
  color: red;
}

因为封装了，它是web组件的点，但你可以用把刀穿过幽灵般的DOM树，像这样：

html /deep/ .braaaain {
  color: red;
}

这虽然只能作用在Blink，如果这是标准那我也没办法。啊，跨浏览器兼容性太可怕了。
